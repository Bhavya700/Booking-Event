# AWS Secrets Manager Integration Guide

This guide explains how to use AWS Secrets Manager with External Secrets Operator to securely manage secrets in your EKS cluster.

## 🔐 Overview

Instead of storing secrets directly in Kubernetes (in `k8s/02-secrets.yaml`), we now use:

1. **AWS Secrets Manager** - Secure storage for sensitive data in AWS
2. **External Secrets Operator** - Syncs secrets from AWS to Kubernetes
3. **IRSA (IAM Roles for Service Accounts)** - Secure authentication without credentials

## 📋 Architecture

```
AWS Secrets Manager
       ↓
External Secrets Operator (with IRSA)
       ↓
Kubernetes Secrets
       ↓
Your Pods
```

## 🚀 Quick Setup

### Step 1: Create Secrets in AWS Secrets Manager

```bash
./scripts/secrets/setup-secrets-manager.sh
```

This script:
- Generates secure random passwords
- Creates `bookmyevent/database` secret (PostgreSQL credentials)
- Creates `bookmyevent/application` secret (JWT & API keys)
- Creates IAM policy for reading these secrets
- Creates IRSA role for External Secrets Operator

### Step 2: Deploy External Secrets Operator

```bash
./scripts/secrets/deploy-external-secrets.sh
```

This script:
- Installs External Secrets CRDs
- Deploys External Secrets Operator
- Creates SecretStore (connection to AWS)
- Creates ExternalSecrets (syncs specific secrets)

### Step 3: Verify Setup

```bash
# Check if External Secrets are synced
kubectl get externalsecrets -n bookmyevent

# Should show:
# NAME                     STATUS   SYNCED
# database-credentials     Valid    True
# application-secrets      Valid    True

# Check if Kubernetes secrets are created
kubectl get secrets -n bookmyevent

# Should show:
# bookmyevent-secrets      Opaque   5      1m
# bookmyevent-app-secrets  Opaque   2      1m
```

## 🔄 How It Works

1. **External Secrets Operator** runs in your cluster with IRSA permissions
2. Every **1 hour**, it fetches secrets from AWS Secrets Manager
3. It creates/updates Kubernetes secrets automatically
4. Your pods use these Kubernetes secrets (no code changes needed!)

## 🛠️ Manual Operations

### View Secrets in AWS

```bash
# List all secrets
aws secretsmanager list-secrets --region us-east-1

# Get database secret value
aws secretsmanager get-secret-value \
  --secret-id bookmyevent/database \
  --region us-east-1 \
  --query SecretString \
  --output text | jq .

# Get application secret value
aws secretsmanager get-secret-value \
  --secret-id bookmyevent/application \
  --region us-east-1 \
  --query SecretString \
  --output text | jq .
```

### Update Secrets in AWS

```bash
# Update database password
aws secretsmanager update-secret \
  --secret-id bookmyevent/database \
  --secret-string '{
    "username": "postgres",
    "password": "new-secure-password",
    "user_service_db_url": "postgresql://postgres:new-secure-password@postgres:5432/users_db?sslmode=disable",
    "event_service_db_url": "postgresql://postgres:new-secure-password@postgres:5432/events_db?sslmode=disable",
    "booking_service_db_url": "postgresql://postgres:new-secure-password@postgres:5432/bookings_db?sslmode=disable"
  }' \
  --region us-east-1

# Wait for sync (up to 1 hour) or force refresh
kubectl delete externalsecret database-credentials -n bookmyevent
kubectl apply -f k8s/secrets-management/03-external-secrets.yaml
```

### Rotate JWT Secret

```bash
# Generate new JWT secret
NEW_JWT_SECRET=$(openssl rand -base64 64 | tr -d "=+/")

# Update in AWS
aws secretsmanager update-secret \
  --secret-id bookmyevent/application \
  --secret-string "{
    \"jwt_secret\": \"$NEW_JWT_SECRET\",
    \"internal_api_key\": \"<existing-api-key>\"
  }" \
  --region us-east-1

# Force refresh
kubectl delete externalsecret application-secrets -n bookmyevent
kubectl apply -f k8s/secrets-management/03-external-secrets.yaml

# Restart services to pick up new secret
kubectl rollout restart deployment/user-service -n bookmyevent
kubectl rollout restart deployment/event-service -n bookmyevent
kubectl rollout restart deployment/booking-service -n bookmyevent
kubectl rollout restart deployment/search-service -n bookmyevent
```

## 🔍 Troubleshooting

### ExternalSecret shows "SecretSyncError"

```bash
# Check ExternalSecret status
kubectl describe externalsecret database-credentials -n bookmyevent

# Common issues:
# 1. IRSA role not configured correctly
# 2. Secret doesn't exist in AWS Secrets Manager
# 3. IAM policy doesn't allow access
```

### Verify IRSA Configuration

```bash
# Check service account annotation
kubectl get sa external-secrets-sa -n bookmyevent -o yaml

# Should show:
# annotations:
#   eks.amazonaws.com/role-arn: arn:aws:iam::{ACCOUNT}:role/BookMyEventExternalSecretsRole

# Check pod has AWS credentials
kubectl exec -it deployment/external-secrets -n bookmyevent -- env | grep AWS
```

### Test IAM Permissions

```bash
# Get pod name
POD=$(kubectl get pod -n bookmyevent -l app=external-secrets -o jsonpath='{.items[0].metadata.name}')

# Test secret access
kubectl exec -it $POD -n bookmyevent -- sh -c '
  aws secretsmanager get-secret-value \
    --secret-id bookmyevent/database \
    --region us-east-1
'
```

## 🔒 Security Best Practices

### DO's

-  Use AWS Secrets Manager for all sensitive data
-  Rotate secrets regularly (every 90 days)
-  Use separate secrets for dev/staging/prod
-  Enable AWS Secrets Manager rotation policies
-  Monitor secret access via CloudTrail
-  Use least-privilege IAM policies

### DON'T

-  Store secrets in Git (ever!)
-  Share secrets across environments
-  Grant broad IAM permissions
-  Disable secret versioning
-  Skip secret rotation

## 🔄 Migration from Kubernetes Secrets

If you're migrating from `k8s/02-secrets.yaml`:

1. **Run setup script first:**
   ```bash
   ./scripts/secrets/setup-secrets-manager.sh
   ```

2. **Deploy External Secrets:**
   ```bash
   ./scripts/secrets/deploy-external-secrets.sh
   ```

3. **Delete old manual secret (optional):**
   ```bash
   kubectl delete secret bookmyevent-secrets -n bookmyevent
   ```
   
   > Note: ExternalSecret will recreate it automatically

4. **Verify services still work:**
   ```bash
   kubectl get pods -n bookmyevent
   kubectl logs deployment/user-service -n bookmyevent
   ```
